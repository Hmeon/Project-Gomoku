# Battle_Omok_AI/docs

이 디렉터리는 `Battle_Omok_AI/` 구현을 **논문/리포트 작성 관점**에서 다시 정리한 문서 모음입니다.  
프로젝트를 처음 실행하는 경우에는 먼저 루트 `README.md`를, 학습/평가 파이프라인을 깊게 다루려면 아래 문서를 권장합니다.

## 문서 목록

- `architecture.md`  
  시스템 구성(게임 루프/룰 엔진/검색/모델)과 모듈 간 데이터 흐름을 설명합니다.

- `renju_rules.md`  
  Renju의 핵심 규칙(흑 금수, 승리 조건)과 본 프로젝트의 구현 원칙을 설명합니다.

- `training.md`  
  self-play -> PV 학습 -> auto-train 반복 루프를 재현 가능한 형태로 정리합니다. 권장 하이퍼파라미터/스케줄도 포함합니다.

- `logging.md`  
  학습이 강화되고 있음을 증명하기 위한 **지표/로그 설계**를 설명합니다. 논문/최종 보고서에서 바로 인용할 수 있는 형태로 CSV 스키마를 포함합니다.

- `plotting.md`  
  로그 CSV/JSON을 기반으로 보고서용 PNG 그래프를 생성하는 간단한 플로팅 절차를 정리합니다.

- `requirements.md`  
  실행/개발 환경 요구사항 및 기능 요구사항(Renju 기반)을 정리합니다.

- `changelog.md`  
  변경 이력 요약입니다(상세는 루트 `PATCH_NOTE_v1.md` 참고).
